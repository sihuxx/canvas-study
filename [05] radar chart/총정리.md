# 🕸️ Canvas 레이더(다각형) 차트 핵심 정리

---

## 1. 다른 차트와 핵심 차이

| | 파이 | 로즈 | 레이더 |
|--|------|------|--------|
| 그리기 | `arc` | `arc` | `lineTo` |
| 각도 | 값에 따라 다름 | 균등 | 균등 |
| 반지름 | 고정 | 값에 따라 다름 | 값에 따라 다름 |
| 핵심 재료 | `total` | `max` | `max` |
| 닫기 | 자동 | 자동 | `closePath()` 필수 |

## 2. 필수 재료

```js
const max = Math.max(...data.map(d => d.v))  // 반지름 비율 계산용
const cx = canvas.width / 2    // 원 중심 x
const cy = canvas.height / 2   // 원 중심 y
const radius = 150             // 최대 반지름
```

---

## 3. 핵심 로직 - 꼭짓점 좌표 계산

레이더 차트의 전부이오. 이것만 이해하면 됨!

```js
function getPoint(i, r) {
  const angle = (Math.PI * 2 / data.length) * i - Math.PI / 2
  //             ↑ 균등하게 나누기               ↑ 12시 방향 보정
  return {
    x: cx + Math.cos(angle) * r,  // 각도 → x 좌표
    y: cy + Math.sin(angle) * r,  // 각도 → y 좌표
  }
}
```

```
data.length = 6 이면

i=0 → angle = 0   - 90 = -90도  → 12시
i=1 → angle = 60  - 90 = -30도  → 2시
i=2 → angle = 120 - 90 =  30도  → 4시
i=3 → angle = 180 - 90 =  90도  → 6시
...
```

## 4. 다각형 그리기 함수

배경 격자랑 데이터 둘 다 이 패턴으로 그림

```js
function drawPolygon(r, fillColor, strokeColor) {
  ctx.beginPath()
  data.forEach((_, i) => {
    const { x, y } = getPoint(i, r)
    i === 0 ? ctx.moveTo(x, y) : ctx.lineTo(x, y)
    //         첫 점은 이동       나머지는 선으로 연결
  })
  ctx.closePath()  // ← 마지막 점 → 첫 점 자동 연결 (필수!)
  if (fillColor) { ctx.fillStyle = fillColor; ctx.fill() }
  if (strokeColor) { ctx.strokeStyle = strokeColor; ctx.stroke() }
}
```

> `closePath()` 없으면 다각형이 열린 채로 남아있소!

## 5. render() 구조 - 5단계

```js
function render(progress) {
  ctx.clearRect(0, 0, canvas.width, canvas.height)

  // ① 배경 격자 (25%, 50%, 75%, 100% 4겹)
  ;[0.25, 0.5, 0.75, 1].forEach(scale => {
    drawPolygon(radius * scale, "rgba(200,200,200,0.15)", "#ddd")
  })

  // ② 중심 → 꼭짓점 축선
  data.forEach((_, i) => {
    const { x, y } = getPoint(i, radius)
    ctx.beginPath()
    ctx.moveTo(cx, cy)  // 중심에서
    ctx.lineTo(x, y)    // 꼭짓점까지
    ctx.strokeStyle = "#ddd"
    ctx.stroke()
  })

  // ③ 데이터 다각형
  ctx.beginPath()
  data.forEach(({ v }, i) => {
    const r = (v / max) * radius * progress  // 값 비율 * progress
    const { x, y } = getPoint(i, r)
    i === 0 ? ctx.moveTo(x, y) : ctx.lineTo(x, y)
  })
  ctx.closePath()
  ctx.fillStyle = "rgba(74, 144, 217, 0.3)"
  ctx.fill()
  ctx.strokeStyle = "#4A90D9"
  ctx.stroke()

  // ④ 데이터 점 (각 꼭짓점에 원)
  data.forEach(({ v }, i) => {
    const r = (v / max) * radius * progress
    const { x, y } = getPoint(i, r)
    ctx.beginPath()
    ctx.arc(x, y, 4, 0, Math.PI * 2)
    ctx.fillStyle = "#4A90D9"
    ctx.fill()
  })

  // ⑤ 라벨 (꼭짓점 바깥쪽)
  data.forEach(({ k }, i) => {
    const { x, y } = getPoint(i, radius + 25)  // radius보다 25px 바깥
    ctx.fillStyle = "#333"
    ctx.textAlign = "center"
    ctx.textBaseline = "middle"
    ctx.fillText(k, x, y)
  })
}
```

## 6. 애니메이션

반지름에 progress 곱하면 끝!

```js
const r = (v / max) * radius * progress
//         값 비율          ↑ 여기에 곱함
```

## 7. closePath vs stroke 차이

```js
ctx.closePath()  // 마지막 점 → 첫 점을 선으로 연결 (경로만 닫음)
ctx.stroke()     // 실제로 화면에 선을 그림
// 둘 다 써야 다각형이 완전히 닫히고 보임!
```

## 8. 전체 흐름 요약

```
getPoint(i, r)
  └─ 균등 각도 * i - 90도 → cos/sin → 좌표 반환

drawPolygon(r)
  └─ getPoint로 꼭짓점 좌표 → moveTo/lineTo → closePath

render(progress)
  ├─ ① 배경 격자: drawPolygon 4번
  ├─ ② 축선: 중심 → 각 꼭짓점
  ├─ ③ 데이터: (v/max)*radius*progress → 꼭짓점 연결
  ├─ ④ 점: 각 꼭짓점에 arc
  └─ ⑤ 라벨: radius+25 위치에 fillText
```

---

## 9. 자주 하는 실수

| 실수 | 결과 | 해결 |
|------|------|------|
| `closePath()` 누락 | 다각형이 열려있음 | forEach 끝나고 `closePath()` |
| `-Math.PI / 2` 누락 | 3시 방향에서 시작 | angle에서 빼주기 |
| 라벨을 `radius`로 계산 | 라벨이 꼭짓점에 겹침 | `radius + 25` 로 바깥에 표시 |
| 배경격자 `forEach` 안에서 `stroke` | 선이 겹쳐서 진해짐 | `drawPolygon` 함수로 분리 |
