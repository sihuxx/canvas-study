# 📊 Canvas 차트 총정리

---

## 1. 4가지 차트 한눈에 비교

| | 바 그래프 | 라인 차트 | 파이 그래프 | 로즈 차트 |
|--|----------|----------|-----------|---------|
| 그리기 메서드 | `fillRect()` | `moveTo` + `lineTo` + `stroke()` | `arc()` + `fill()` | `arc()` + `fill()` |
| 핵심 재료 | `sectionWidth` | `sectionWidth` | `total` | `max` |
| 값 반영 | 막대 높이 | y 좌표 | 각도 | 반지름 |
| 각도 | 없음 | 없음 | 값에 따라 다름 | 전부 균등 |
| 반지름 | 없음 | 없음 | 고정 | 값에 따라 다름 |
| progress 곱하는 곳 | height | y 좌표 | angle | r (반지름) |

---

## 2. 공통 재료

```js
const canvas = document.querySelector('canvas')
const ctx = canvas.getContext('2d')
const duration = 1000   // 애니메이션 시간
let startTime
```

---

## 3. 애니메이션 패턴 (모든 차트 공통!)

```js
function animate(timestamp) {
  if (!startTime) startTime = timestamp
  const elapsed = timestamp - startTime
  const progress = Math.min(elapsed / duration, 1)  // 0 → 1
  const ease = 1 - (1 - progress) ** 3              // easeOutCubic
  render(ease)
  if (progress < 1) requestAnimationFrame(animate)
}
requestAnimationFrame(animate)
```

> `render()` 안에서 progress를 **어디에 곱하냐**만 다르고 나머지는 전부 동일!

---

## 4. 이징(Easing)

| 이름 | 수식 | 느낌 |
|------|------|------|
| linear | `progress` | 일정한 속도 |
| easeOutQuad | `1 - (1 - progress) ** 2` | 빠르게 시작 → 천천히 끝 |
| easeOutCubic | `1 - (1 - progress) ** 3` | 더 강하게 감속 |
| easeInQuad | `progress ** 2` | 천천히 시작 → 빠르게 끝 |
| easeInOut | `progress < 0.5 ? 2 * progress ** 2 : 1 - (-2 * progress + 2) ** 2 / 2` | S커브 |

## 5. 바 그래프

### 핵심 재료
```js
const sectionWidth = (canvas.width - padding) / data.length
const max = Math.max(...data)
```

### 핵심 패턴
```js
data.forEach((value, i) => {
  const x = sectionWidth * i + gap       // x + width = sectionWidth*(i+1) 이 되게!
  const width = sectionWidth - gap
  const height = valueToHeight(value) * progress  // progress 곱하는 곳!

  ctx.fillRect(x, canvas.height - padding, width, -height)  // 음수 = 위로
})
```

### 중요 포인트
- `x + width = sectionWidth * (i+1)` 이 공식만 맞으면 정렬 완성
- `fillRect` height 음수면 위로 그려짐
- `clearRect` 없으면 프레임마다 겹쳐서 진해짐

## 6. 라인 차트

### 핵심 재료
```js
const sectionWidth = (canvas.width - padding) / data.length
const max = Math.max(...data.map(d => d.value))
```

### 핵심 패턴
```js
// 선 그리기 (beginPath 한 번!)
ctx.beginPath()
data.forEach((value, i) => {
  const x = padding + sectionWidth * i + sectionWidth / 2
  const y = canvas.height - padding - valueToHeight(value) * progress
  i === 0 ? ctx.moveTo(x, y) : ctx.lineTo(x, y)
})
ctx.stroke()

// 점 그리기 (forEach 따로!)
data.forEach((value, i) => {
  ctx.beginPath()
  ctx.arc(x, y, 5, 0, Math.PI * 2)
  ctx.fill()
})
```

### 중요 포인트
- `beginPath()` 한 번 → `stroke()` 한 번 (선 전체)
- 점은 **반드시 forEach 따로** 돌려야 선 경로가 안 끊김
- `moveTo` = 펜 들고 이동 / `lineTo` = 펜 대고 이동

## 7. 파이 그래프

### 핵심 재료
```js
const total = data.reduce((acc, cur) => acc + cur.v, 0)  // 전체 합!
const cx = canvas.width / 2
const cy = canvas.height / 2
const radius = 200
```

### 핵심 패턴
```js
let current = -Math.PI / 2  // 12시 방향 시작

data.forEach(({ v, c }, i) => {
  const angle = (v / total) * Math.PI * 2  // 값 → 각도 변환

  ctx.beginPath()
  ctx.moveTo(cx, cy)
  ctx.arc(cx, cy, radius, current, current + angle * progress)  // progress 곱하는 곳!
  ctx.fillStyle = c
  ctx.fill()

  current += angle * progress  // current도 같이 progress 곱하기!
})
```

### 라벨 위치 (cos, sin)
```js
const midAngle = current + (angle * progress) / 2
const lx = cx + Math.cos(midAngle) * (radius * 0.65)  // 각도 → 좌표
const ly = cy + Math.sin(midAngle) * (radius * 0.65)
```

### 호버 이벤트
```js
// 원 안에 있는지 (피타고라스)
Math.sqrt(dx**2 + dy**2) > radius  // true면 원 밖

// 몇 번째 조각인지 (atan2 = 좌표 → 각도)
let mouseAngle = Math.atan2(dy, dx)
if (mouseAngle < -Math.PI / 2) mouseAngle += Math.PI * 2  // 음수 보정
```

### 중요 포인트
- `total` 로 각도 비율 계산 (로즈는 `max`)
- `current` 누적 안 하면 모든 조각이 겹침
- `current`에도 `* progress` 곱해야 조각이 안 벌어짐
- 시작각도 `-Math.PI / 2` = 12시 (캔버스 y축이 아래로 증가해서 뒤집힘)

## 8. 로즈 차트

### 핵심 재료
```js
const max = Math.max(...data.map(d => d.v))  // 최댓값! (파이는 total)
const angle = Math.PI * 2 / data.length      // 균등한 고정 각도
```

### 핵심 패턴
```js
let current = -Math.PI / 2

data.forEach(({ v, c }) => {
  const r = (v / max) * radius * progress  // 반지름이 값에 따라 다름! progress 곱하는 곳

  ctx.beginPath()
  ctx.moveTo(cx, cy)
  ctx.arc(cx, cy, r, current, current + angle * progress)  // 각도도 progress
  ctx.fillStyle = c
  ctx.fill()

  current += angle * progress
})
```

### 중요 포인트
- 파이와 딱 두 줄만 다름: `angle` 계산, `r` 계산
- 라벨은 `radius * 0.65` 아닌 **`r * 0.65`** (조각마다 반지름이 다름!)
- `max` 로 반지름 비율 계산 (파이는 `total`)

## 9. 공통 중요 메서드 총정리

| 메서드 | 용도 |
|--------|------|
| `ctx.clearRect(0,0,w,h)` | 매 프레임 화면 초기화 (필수!) |
| `ctx.fillRect(x,y,w,h)` | 사각형 그리기 |
| `ctx.beginPath()` | 새 경로 시작 |
| `ctx.moveTo(x,y)` | 펜 들고 이동 (선 없음) |
| `ctx.lineTo(x,y)` | 펜 대고 이동 (선 있음) |
| `ctx.stroke()` | 경로를 선으로 그리기 |
| `ctx.arc(x,y,r,s,e)` | 원/호 그리기 |
| `ctx.fill()` | 경로 채우기 |
| `ctx.globalAlpha` | 투명도 (반드시 1로 초기화!) |
| `Math.cos(angle)` | 각도 → x 좌표 방향 |
| `Math.sin(angle)` | 각도 → y 좌표 방향 |
| `Math.atan2(dy,dx)` | 좌표 → 각도 (호버에 사용) |
| `Math.sqrt(dx**2+dy**2)` | 두 점 사이 거리 (피타고라스) |

---

## 10. 자주 하는 실수 총정리

| 실수 | 해결 |
|------|------|
| `clearRect` 누락 | 매 render 첫줄에 추가 |
| `stroke()` 누락 | forEach 끝나고 호출 |
| `beginPath()` 누락 | 선/원 시작 전에 호출 |
| 선이랑 점 같은 forEach | forEach 분리 |
| `current` 누적 안 함 | `current += angle * progress` |
| `current`에 progress 안 곱함 | 조각이 벌어짐 |
| `globalAlpha` 초기화 안 함 | 텍스트도 흐려짐 |
| `total` 대신 `max` (파이) | 각도 계산 오류 |
| `max` 대신 `total` (로즈) | 반지름 계산 오류 |
| 라벨 `textAlign` 나중에 설정 | 정렬 안 됨 |
