# 📈 Canvas 라인 차트 핵심 정리

---

## 1. 바 그래프와 핵심 차이

| | 바 그래프 | 라인 차트 |
|--|--------|--------|
| 그리기 | `fillRect()` | `beginPath()` + `lineTo()` + `stroke()` |
| 좌표 | x, y, width, height | x, y 점만 |
| 점 표시 | 없음 | `arc()`로 원 추가 |
| `beginPath` 위치 | 막대마다 각각 | 전체 선 한 번만 |

---

## 2. 라인 차트 그리기 핵심 패턴

```js
ctx.beginPath()  // ← 전체 선에 한 번만!
data.forEach((value, i) => {
  const x = padding + sectionWidth * i + sectionWidth / 2  // 구역 중앙
  const y = canvas.height - padding - valueToHeight(value)

  if (i === 0) ctx.moveTo(x, y)  // 첫 점: 펜 올리기
  else ctx.lineTo(x, y)          // 나머지: 선 긋기
})
ctx.stroke()  // ← 반드시 마지막에 한 번!
```

> `beginPath()`가 없으면 이전 경로와 이어져 버림
> `stroke()`가 없으면 아무것도 안 그려짐

---

## 3. 점(원) 추가

```js
// forEach를 따로 돌려야 함 (beginPath가 선과 충돌)
data.forEach((value, i) => {
  const x = padding + sectionWidth * i + sectionWidth / 2
  const y = canvas.height - padding - valueToHeight(value)

  ctx.beginPath()              // 원마다 새로운 경로!
  ctx.arc(x, y, 5, 0, Math.PI * 2)  // (x, y, 반지름, 시작각, 끝각)
  ctx.fill()
})
```

> 선이랑 같은 forEach에 넣으면 선 경로가 끊김. **반드시 따로 돌리기!**

---

## 4. arc() 메서드

```js
ctx.arc(x, y, radius, startAngle, endAngle)
//      중심x, 중심y, 반지름, 시작각도, 끝각도

ctx.arc(x, y, 5, 0, Math.PI * 2)  // 완전한 원
ctx.arc(x, y, 5, 0, Math.PI)      // 반원
```

## 5. 단축 문법 팁

```js
// 이거나
if (i === 0) ctx.moveTo(x, y)
else ctx.lineTo(x, y)

// 이거나 같음
ctx[i === 0 ? 'moveTo' : 'lineTo'](x, y)
```

## 6. 이징(Easing) - 애니메이션 부드럽게

```js
function animate(timestamp) {
  if (!startTime) startTime = timestamp
  const elapsed = timestamp - startTime
  const progress = Math.min(elapsed / duration, 1)  // 종료 판단은 원본 progress로!

  const eased = 1 - (1 - progress) ** 3  // easeOutCubic
  render(eased)  // render엔 eased를 넘김

  if (progress < 1) requestAnimationFrame(animate)
}
```

| 이름 | 수식 | 느낌 |
|------|------|------|
| linear | `progress` | 일정한 속도 |
| easeOutQuad | `1 - (1 - progress) ** 2` | 빠르게 시작 → 천천히 끝 |
| easeOutCubic | `1 - (1 - progress) ** 3` | 더 강하게 감속 |
| easeInQuad | `progress ** 2` | 천천히 시작 → 빠르게 끝 |
| easeInOut | `progress < 0.5 ? 2 * progress ** 2 : 1 - (-2 * progress + 2) ** 2 / 2` | S커브 |

> 라인차트엔 **easeOutCubic** 이 제일 자연스럽소!

## 7. 라벨 텍스트

```js
ctx.textAlign = "center"           // 중앙 정렬 (fillText 전에 설정!)
ctx.fillText(label, x, y + 40)     // 점 아래에 텍스트
```

> `textAlign`은 `fillText` **이전에** 설정해야 적용되오!

---

## 8. 자주 하는 실수

| 실수 | 결과 | 해결 |
|------|------|------|
| `stroke()` 누락 | 아무것도 안 그려짐 | forEach 끝나고 `stroke()` 호출 |
| 선이랑 원 같은 forEach | 선 경로가 끊김 | forEach 분리 |
| `beginPath()` 누락 | 이전 경로와 이어짐 | 선 시작 전 `beginPath()` |
| y값 고정 | 모든 점이 바닥에 | value 기반으로 y 계산 |
| `textAlign` 나중에 설정 | 정렬 안 됨 | `fillText` 전에 설정 |
